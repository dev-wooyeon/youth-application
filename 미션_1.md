# 1. 학습 과정과 현재 자신이 생각하는 역량은 무엇인가요? (1000자 이내)

**[학습과정]**  
저는 **야생형**으로 배우는 사람입니다.  
모르면 당장 해봐야 직성이 풀리고, 알게 되면 기반이 되는 이론을 학습해 나갑니다.  
기본적으로 기초는 책으로 다지고, 실무에서 바로 써야 하는 건 온라인 강의로 상황에 맞게 해결합니다.  
배운 건 꼭 제 언어로 정리하고, 제 생각을 덧붙여 기록으로 남깁니다. 나중에 다시 꺼내 봐도 ‘왜 이렇게 생각했는지’가 보이도록 노력합니다.  
독서 : https://github.com/dev-wooyeon/reading-books-for-programmers  
블로그 : https://ieunune.notion.site/velog-573ade80268f45aeae75b09c5b86ac02  

**[역량]**  
제가 가진 대표 역량은 두 가지입니다.

첫째는 **단순화** 능력입니다.
복잡한 로직을 쉽게 읽히는 구조로 바꾸고, 흐름의 핵심만 남겨 팀원들이 빠르게 이해할 수 있도록 정리하는 걸 좋아합니다.
정산 업무 자동화 과정에서도 비지니스 로직을 처음부터 다시 구조화해 운영자들의 하루 업무 시간을 2시간 이상 줄인 경험이 있습니다.

둘째는 **자동화**와 **효율화** 역량입니다.
반복되거나 휴먼 에러가 발생하기 쉬운 작업은 자동화하여 팀 생산성을 높이는 걸 중요하게 여깁니다.
Python 업무가 필요할 때 1주일 만에 기본기를 익히고 Sanic 기반 신규 API 개발을 진행했던 경험처럼, 특정 기술에 얽매이지 않고 필요한 도구를 빠르게 흡수합니다.

리더십은 지급대행 프로젝트에서 처음 경험했습니다.
3명 규모의 프로젝트였지만 일정 조율, 기술 스택 선정(Spring Boot, JPA, 테스트 코드 도입) 등 팀의 기술 기반을 세우며 협업의 중요성과 실행의 가치를 배웠습니다.

지난 몇 년 동안 시행착오를 겪으며 누군가 저와 같은 고민을 할 누군가에게 도움이 되고자 멘토링을 하였습니다. 지금까지 100명 넘는 취준생과 주니어 개발자들과 함께했고, 과정에서 제가 더 많이 배우기도 하였습니다.

결국 제 역량의 중심엔 항상 실행력이 있습니다.
모르면 배우고, 알면 개선하고, 익숙해지면 자동화합니다.
복잡한 것을 단순하게 만들고, 불편한 것을 효율적으로 바꾸는 일을 가장 잘하는 것 같습니다.

# 2. 선택한 직군을 목표로 하게 된 이유는 무엇인가요? (1000자 이내)

**[과거 경험]**  
제가 지금의 직군을 목표로하게 된 이유는 단순히 ‘개발로 사람들의 편리함을 가져다 주는 것’에서 시작했지만 결국 ‘데이터를 잘 다루는 개발자’로 성장하고 싶다는 목표로 이어졌기 때문입니다.  

정산팀에서 3년간 일하며 매일 수만 건의 거래 데이터를 다뤘습니다.  
처음엔 단순히 정합성을 맞추는 일이었지만, 어느 순간부터 ‘이 데이터를 더 좋은 방향으로 만들순 없었을까?’라는 고민으로부터 시작했습니다.  

배치 자동화를 구축하고, 오류 케이스를 분류하며 시스템 안정성을 높이면서 깨달은 건 하나였습니다.  
백엔드 로직보다 더 본질적인 건 데이터의 구조와 흐름이라는 점이었습니다.  
이후 테마파크 IoT 팀으로 이직하며 센서, 하드웨어 등의 데이터를 실시간으로 수집하고 가공되어 최종적으로 분석 테이블로 쌓이는 과정을 경험했습니다.

**[데이터 엔지니어를 목표로 하게된 이유]**  
서버 개발자 입장에서 데이터를 저장하고 꺼내는 수준을 넘어, ‘데이터의 생애주기’를 설계하는 관점을 배우게 되었습니다. 이 경험은 자연스럽게 저를 데이터 엔지니어의 관심을 가지게 되는 계기가 되었습니다.
저는 여전히 백엔드 엔지니어이지만, 제 시선은 데이터 엔지니어의 일에 닿아 있습니다. 

Shell Script, Spark, DuckDB를 활용하여 1BRC(10억행챌린지)를 통해 대용량 데이터를 처리할 수 있는 파이프라인을 직접 설계해보았습니다. 
단순 데이터만 15GB가 넘는 수준이였는데 수분이내에 ETL하여 가치 있는 데이터를 만들어내는 것에 매력을 느렸습니다.

시스템을 단순히 안정적으로 “운영하는 사람”이 아니라, 데이터를 통해 “의미를 추출하고 개선 방향을 제시하는 사람”이 되고 싶습니다.
저는 시스템 구조와 데이터 흐름을 설계하는 데 강점이 있습니다. 업무에서 반복되는 패턴을 자동화하고, 사람의 개입을 줄이는 구조를 만드는걸 즐깁니다. 
그리고 그 과정에서 효율성뿐 아니라 ‘이 데이터가 나중에 어떻게 쓰일 수 있을까’를 항상 고려합니다.
결국 제가 이 직군을 목표로 하는 이유는 명확합니다. 
데이터 엔지니어는 제가 그동안 쌓아온 정확성과 효율성 중심의 백엔드 경험을 가장 잘 확장할 수 있는 영역이기 때문입니다. 시스템의 안정성을 다루던 사람이 이제는 데이터의 가치를 다루는 사람으로 성장하고자 합니다.

# 3. 마음이 끌리는 무언가에 몰입하여 일정 수준의 결과물을 만들어 낸 경험이 있나요? (1000자 이내)

**[과거 몰입 경험]**  
저는 **10년간 해동검도를 수련**한 경험이 있습니다. 처음 도장을 찾은 이후로 학업과 병행하면서도 단 한 번도 연습을 중단한 적이 없습니다.
지루함보다는 ‘조금이라도 더 나아지고 싶다’라는 마음이 강했고 이 꾸준함 덕분에 **유단자 4단**을 취득하고, **세계대회 본선**에 진출하는 성과를 거둘 수 있었습니다.
3단을 취득한 이후에는 단순히 기술을 익히는 단계에서 벗어나 모범의 역할을 맡았습니다.

처음엔 지도보다 시범에 집중했지만, 점차 후배 성장 속도를 이끌기 위해 수련 계획을 직접 짜고, 시연 동선을 설계하며 팀을 리드하는 경험을 쌓았습니다.
특히 대회 준비 과정에서는 팀 전체의 컨디션과 호흡을 고려해야 했기에, ‘혼자 잘하는 사람’이 아니라 ‘함께 완성하는 리더’로 변화할 수 있었습니다.
이 경험을 통해 인내심, 책임감, 문제 해결력의 중요성을 체득했습니다.

검도를 통해 얻은 가장 큰 배움은 ‘꾸준함은 재능을 이긴다’는 믿음이었습니다.
몸이 힘들 때도 꾸준히 수련을 이어가며 촛불끄기, 종이베기, 공베기의 결과가 성장하는 과정 자체가 제게는 가장 큰 즐거움이었습니다.
이 경험은 이후 제가 새로운 기술이나 프로젝트를 배우는 태도에도 그대로 이어졌습니다.

**[최근 몰입 경험]**  
최근에는 이러한 몰입 습관을 개발 분야로 확장했습니다.  
AI 에이전트인 Codex와 AWS Kiro를 활용해 좋아하는 게임의 공지사항을 자동으로 감지해 알림을 보내는 서비스를 직접 설계하고 구축하여 배포했습니다.  
혼자 1주일간 기획부터 개발, 배포까지 진행하며 실험과 개선을 반복한 경험은, 도장에서 한 동작을 끊임없이 연습하던 시간과 닮아 있었습니다.  
링크 : https://github.com/dev-wooyeon/kiki

저에게 몰입이란 단순히 열중하는 것이 아니라 결과가 보일 때까지 포기하지 않는 꾸준함의 다른 이름입니다. 
10년간의 수련과 프로젝트 경험을 통해 저는 끝까지 해내는 힘을 몸에 익혔고, 앞으로도 이런 태도로 새로운 환경에서도 흔들리지 않고 성장해 나가겠습니다.


# 4. 우리 회사에 지원하려는 이유는 무엇인가요? (500자 이내)

대상 채용 공고 : https://toss.im/career/job-detail?job_id=7450926003

**[과거 경험]**  
정산팀에서 3년 동안 월 백만 건 수준의 거래 데이터를 다루며 정확한 계산을 바탕으로 정산 대금을 지급하고 반복 업무를 자동화하며 데이터 흐름을 개선해 운영 효율을 높인 경험이 있습니다. 
이 경험은 토스뱅크 CPC가 지향하는 신뢰도 높은 데이터 체계 구축과 자동화, 고도화 업무에 즉시 기여할 수 있는 부분이라고 생각합니다.

**[요구 역량 경험]**  
결제자가 판매자로부터 환불을 받지 못하는 경우 카드사에서 차지백 데이터를 수기로 전달하는 프로세스가 있었고, 초기에는 운영자가 수동으로 엑셀을 등록해 차감 처리까지 수행/해야 했습니다. 
반복 업무로 인한 리소스 소모가 커 자동화를 통해 해결한 경험이 있습니다.  
또한 일부 가맹점에서 부가세 참고자료 조회가 느려지는 문제를 Pinpoint APM을 활용하여 원인을 분석하고 불필요한 LEFT JOIN 제거와 복합 인덱스 적용을 통해 근본적으로 해결한 경험도 있습니다.

**[기술적 역량]**  
아직 데이터 엔지니어링 영역을 깊게 경험한 것은 아니지만,  
10억 행 챌린지를 통해 대규모 데이터 기반 성능 분석과 다양한 오픈소스 실험을 진행하며 꾸준히 역량을 쌓아가고 있습니다.  
실무에서 대규모 데이터를 통해 가치를 만들어내는 일에 대한 갈증이 크며, 그동안 쌓아온 역량을 실무 환경에서 발휘하고 싶습니다.

이러한 경험과 성장 의지를 바탕으로 토스뱅크 CPC 파트에서 데이터 웨어하우스 팀에 기여하고자 지원했습니다.
